# Схема DTO API Food Helper

Документ описывает форматы запросов и ответов REST API (JSON). Все даты и идентификаторы — в формате, указанном в полях.

---

## 1. Успешные ответы

### 1.1. Список служб доставки

**Эндпоинт:** `GET /api/v1/delivery-services`

**Ответ:** массив объектов `DeliveryService`.

| Поле      | Тип     | Обязательное | Описание                    |
|-----------|---------|--------------|-----------------------------|
| id        | long    | да           | Идентификатор               |
| code      | string  | да           | Код службы (латиница)       |
| name      | string  | да           | Название                    |
| siteUrl   | string? | нет          | URL сайта                   |
| logoUrl   | string? | нет          | URL логотипа                |
| active    | boolean | да           | Признак активности (по умолч. true) |

### 1.2. Поиск продуктов (пагинированный список)

**Эндпоинт:** `GET /api/v1/products`

**Ответ:** объект `ProductPage`.

| Поле           | Тип              | Обязательное | Описание                    |
|----------------|------------------|--------------|-----------------------------|
| items          | array of Product | да           | Элементы страницы           |
| page           | int              | да           | Номер страницы (0-based)    |
| size           | int              | да           | Размер страницы             |
| totalElements  | long             | да           | Общее количество записей    |
| totalPages     | int              | да           | Общее количество страниц    |

**Product:**

| Поле            | Тип                | Обязательное | Описание                    |
|-----------------|--------------------|--------------|-----------------------------|
| id              | long               | да           | Идентификатор продукта      |
| name            | string             | да           | Название                    |
| url             | string             | да           | Ссылка на товар в сервисе   |
| price           | int                | да           | Цена в минимальных единицах |
| currency        | string             | да           | Код валюты (по умолч. RUB)  |
| deliveryService | DeliveryService    | да           | Служба доставки             |
| variants        | array of ProductVariant | нет     | Варианты (объём/состав)      |

**ProductVariant:**

| Поле         | Тип        | Обязательное | Описание           |
|--------------|------------|--------------|--------------------|
| id           | long       | да           | Идентификатор      |
| manufacturer | string?    | нет          | Производитель      |
| composition  | string?    | нет          | Состав             |
| weight       | int?       | нет          | Вес (г)            |
| nutrients    | Nutrients  | да           | КБЖУ варианта      |

**Nutrients:**

| Поле     | Тип    | Обязательное | Описание    |
|----------|--------|--------------|-------------|
| calories | int?   | нет          | Калории     |
| protein  | double?| нет          | Белки (г)   |
| fat      | double?| нет          | Жиры (г)    |
| carbs    | double?| нет          | Углеводы (г)|

## 2. Единый формат ошибок (ErrorResponse)

При любом ответе с ошибкой (4xx, 5xx) тело ответа — JSON одного формата для корреляции в логах и единообразной обработки на клиенте.

| Поле       | Тип              | Обязательное | Описание |
|------------|------------------|--------------|----------|
| requestId  | string           | да           | Идентификатор запроса (UUID). Передавать в заголовке `X-Request-Id` при отладке или использовать значение из ответа для поиска в логах. |
| code       | string           | да           | Код ошибки приложения: `VALIDATION_ERROR`, `BUSINESS_VALIDATION_ERROR`, `NOT_FOUND`, `DB_ERROR`, `INTERNAL_ERROR`. |
| message    | string           | да           | Краткое человекочитаемое описание ошибки. |
| status     | int              | да           | HTTP-статус (400, 404, 422, 500). |
| path       | string           | да           | Путь запроса. |
| timestamp  | string           | да           | Момент времени (ISO-8601). |
| details    | array of string? | нет          | Дополнительные сообщения (например, список нарушений). |
| fieldErrors| array of FieldError? | нет       | Ошибки по полям ввода (для 400/422). |

**FieldError:**

| Поле     | Тип    | Описание                    |
|----------|--------|-----------------------------|
| field    | string | Имя параметра/поля          |
| message  | string | Сообщение об ошибке для поля|

### Примеры ErrorResponse

**400 — неверные параметры запроса (VALIDATION_ERROR):**

```json
{
  "requestId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "code": "VALIDATION_ERROR",
  "message": "Параметры запроса не прошли проверку",
  "status": 400,
  "path": "/api/v1/products",
  "timestamp": "2025-02-21T12:00:00Z",
  "details": ["size must be between 1 and 100"],
  "fieldErrors": [
    { "field": "size", "message": "must be between 1 and 100" }
  ]
}
```

**422 — бизнес-валидация (BUSINESS_VALIDATION_ERROR):**

```json
{
  "requestId": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
  "code": "BUSINESS_VALIDATION_ERROR",
  "message": "Запрос корректен по формату, но не прошёл бизнес-правила",
  "status": 422,
  "path": "/api/v1/...",
  "timestamp": "2025-02-21T12:00:00Z",
  "details": ["Указанная служба доставки недоступна"]
}
```

**500 — внутренняя ошибка (INTERNAL_ERROR):**

```json
{
  "requestId": "c3d4e5f6-a7b8-9012-cdef-123456789012",
  "code": "INTERNAL_ERROR",
  "message": "Внутренняя ошибка сервера",
  "status": 500,
  "path": "/api/v1/products",
  "timestamp": "2025-02-21T12:00:00Z"
}
```

## 3. Соответствие HTTP-статусов и кодов

| HTTP | code                      | Когда использовать |
|------|---------------------------|--------------------|
| 400  | VALIDATION_ERROR          | Неверный формат или значения параметров (тип, диапазон, обязательность). |
| 404  | NOT_FOUND                 | Ресурс не найден. |
| 422  | BUSINESS_VALIDATION_ERROR | Формат запроса верный, но нарушены бизнес-правила (например, несуществующая служба доставки, недопустимая комбинация фильтров). |
| 500  | INTERNAL_ERROR, DB_ERROR  | Внутренняя ошибка или ошибка БД. |

Клиенту достаточно проверять `status` и `code` и при необходимости показывать `message` и `fieldErrors` для подсветки полей ввода.
