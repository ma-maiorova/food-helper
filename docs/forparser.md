# Спецификация для парсера: импорт в Food Helper API

Документ описывает формат запроса и ответа эндпоинта импорта продуктов.

## Эндпоинт

- **Метод:** `POST`
- **URL:** `{baseUrl}/api/v1/admin/import` (например, `http://localhost:8080/api/v1/admin/import`)
- **Content-Type:** `application/json`

## Ограничения на стороне сервиса

- Обрабатывается не более **N** элементов за один запрос (N задаётся в конфиге, по умолчанию 500). Если в `items` больше N, учитываются только первые N.
- Служба доставки с кодом `deliveryServiceCode` должна **уже существовать** в БД (создаётся отдельно через админку или API).
- Продукт в БД определяется по паре **(служба доставки, url)**. Повторная отправка того же `url` для той же службы **обновляет** продукт и варианты (идемпотентность).

## Формат запроса (тело JSON)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `deliveryServiceCode` | string | да | Код службы доставки (должен совпадать с записью в БД, например `VKUSVILL`, `YANDEX_EATS`). |
| `items` | array | да | Массив товаров. Имя поля именно **`items`** (не `products`). |

### Элемент массива `items` (один товар)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `name` | string | да | Название продукта. Не пустое, до 512 символов. |
| `url` | string | да | URL товара. Не пустой, до 1024 символов. Уникален в рамках одной службы доставки. |
| `price` | integer | да | Цена, целое число, ≥ 0. |
| `currency` | string | нет | Валюта, по умолчанию `RUB` (до 8 символов). |
| `id` | number (int64) | нет | Игнорируется сервисом; можно передавать для отладки. |
| `deliveryService` | object | нет | Игнорируется; служба определяется по `deliveryServiceCode`. |
| `variants` | array | нет | Варианты продукта (объём, состав, КБЖУ). По умолчанию пустой массив. |

### Элемент массива `variants` (один вариант)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `manufacturer` | string | нет | Производитель, до 255 символов. |
| `composition` | string | нет | Состав (текст). |
| `weight` | integer | нет | Вес в граммах. |
| `nutrients` | object | нет | Нутриенты. Все поля внутри необязательные. |
| `id` | number (int64) | нет | Игнорируется. |

### Объект `nutrients`

| Поле | Тип | Описание |
|------|-----|----------|
| `calories` | integer | Калории (ккал). |
| `protein` | number (double) | Белки, г. |
| `fat` | number (double) | Жиры, г. |
| `carbs` | number (double) | Углеводы, г. |

**Важно:** нутриенты передаются только внутри объекта **`nutrients`** у варианта, а не на верхнем уровне варианта.

## Пример тела запроса

```json
{
  "deliveryServiceCode": "VKUSVILL",
  "items": [
    {
      "name": "Молоко 3.2% 1л",
      "url": "https://vkusvill.ru/goods/moloko-32-1l.html",
      "price": 89,
      "currency": "RUB",
      "variants": [
        {
          "manufacturer": "ООО Молоко",
          "composition": "молоко нормализованное",
          "weight": 1000,
          "nutrients": {
            "calories": 64,
            "protein": 3.0,
            "fat": 3.2,
            "carbs": 4.7
          }
        }
      ]
    },
    {
      "name": "Хлеб белый нарезной",
      "url": "https://vkusvill.ru/goods/hleb-belyy.html",
      "price": 45,
      "variants": [
        {
          "nutrients": {
            "calories": 265,
            "protein": 7.5,
            "fat": 2.0,
            "carbs": 52.0
          }
        }
      ]
    }
  ]
}
```

## Формат ответа

- **Код ответа:** всегда **200 OK** (ошибки по отдельным элементам не меняют HTTP-код).
- **Тело:** JSON.

| Поле | Тип | Описание |
|------|-----|----------|
| `importedCount` | integer | Количество успешно загруженных или обновлённых продуктов. |
| `failedCount` | integer | Количество элементов, по которым была ошибка или не прошла валидация. |
| `errors` | array | Список ошибок по элементам. |

### Элемент массива `errors`

| Поле | Тип | Описание |
|------|-----|----------|
| `itemIndex` | integer | Индекс элемента в переданном `items` (с нуля). Для общих ошибок (например, служба не найдена) может быть `-1`. |
| `url` | string \| null | URL продукта, по которому ошибка. |
| `name` | string \| null | Название продукта. |
| `message` | string | Текст ошибки (валидация или сообщение от БД). |

## Пример ответа (успех)

```json
{
  "importedCount": 2,
  "failedCount": 0,
  "errors": []
}
```

## Пример ответа (частичная ошибка)

```json
{
  "importedCount": 1,
  "failedCount": 1,
  "errors": [
    {
      "itemIndex": 1,
      "url": "https://vkusvill.ru/goods/invalid.html",
      "name": "Товар с ошибкой",
      "message": "url слишком длинный"
    }
  ]
}
```

## Пример ответа (служба доставки не найдена)

```json
{
  "importedCount": 0,
  "failedCount": 2,
  "errors": [
    {
      "itemIndex": -1,
      "url": null,
      "name": null,
      "message": "Служба доставки с кодом 'UNKNOWN' не найдена"
    }
  ]
}
```

## Рекомендации для парсера

1. Перед импортом убедиться, что служба доставки создана в системе (по коду из `deliveryServiceCode`).
2. Использовать поле **`items`**, не `products`.
3. Нутриенты передавать только в объекте **`nutrients`** внутри каждого варианта.
4. При большом объёме данных можно слать один запрос: сервис сам ограничит число обрабатываемых элементов (по умолчанию 500) и разобьёт их на внутренние батчи.
5. По полям `importedCount`, `failedCount` и `errors` определять, что загрузилось, а что нет, и при необходимости повторять или корректировать данные для элементов из `errors`.
