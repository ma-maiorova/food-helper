# FoodHelper: Поиск готовой еды по КБЖУ

Telegram-бот для поиска готовой еды из популярных сервисов доставки по  значениям КБЖУ (калории, белки, жиры, углеводы).

---
## Поддерживаемые сервисы доставки [в разработке]

* ВкусВилл
* Яндекс Лавка
* Самокат [опционально]

---
## Ключевые возможности [в разработке]

* **Фильтрация по КБЖУ:** Гибкая настройка диапазонов для калорий, белков, жиров и углеводов.
* **Выбор служб доставки:** Поиск по одной или нескольким службам одновременно.
* **Поиск по названию:** Быстрый поиск конкретных продуктов.
* **Понятные результаты:** Карточки товаров с названием, ценой, КБЖУ и прямой ссылкой на товар в приложении доставки.

## Документация API (Swagger / OpenAPI)

REST API сервиса документирован в формате **OpenAPI 3.0**. После запуска приложения доступны:

| Что | URL |
|-----|-----|
| **Swagger UI** (интерактивная документация и вызов запросов) | `GET /swagger` → редирект на UI с подставленной спецификацией |
| **Раздача Swagger UI** | `GET /swagger-ui/` (статика из webjar) |
| **Спецификация OpenAPI (YAML)** | `GET /openapi/openapi.yaml` |

### Как пользоваться

1. Запустите backend (из каталога `service`: `./gradlew run` или через IDE).
2. Откройте в браузере **http://localhost:8080/swagger** (порт уточните в `service/src/main/resources/application.conf`).
3. В Swagger UI отображаются эндпоинты `/api/v1/delivery-services` и `/api/v1/products`, схемы ответов и ошибок (400, 422, 500). Запросы можно выполнять прямо из интерфейса.

Схема DTO и единый формат ошибок (ErrorResponse с `requestId`, `code`, `message`, `details`, `fieldErrors`) описаны в **[docs/DTO_SCHEMA.md](docs/DTO_SCHEMA.md)**.

## Запуск в dev (Docker)

Стабильный dev-режим: Postgres + API, с healthchecks и переменными из `.env`.

```bash
cd service
cp .env.example .env   # при необходимости отредактировать
docker compose --env-file .env up -d
```

- **Postgres:** порт по умолчанию `5435` (или `POSTGRES_PORT` из `.env`), healthcheck — `pg_isready`.
- **API:** порт `8080`, healthcheck — `GET /health`. Зависит от готовности Postgres.
- Позже в тот же `docker-compose` можно добавить сервис **loader**.

Локальный запуск без Docker: поднять только Postgres (`docker compose up -d postgres`), в `.env` указать `DB_JDBC_URL=jdbc:postgresql://127.0.0.1:5435/food_helper`, затем `./gradlew run` в каталоге `service`.
